Web Bluetooth Guide

            
Understanding setup requirements

          

        


        

          

            1. Browser Support: Web Bluetooth is supported in Google Chrome, Edge, and Opera on Windows 10/11, macOS, Linux, and Android. It is not supported in iOS Safari due to Apple restrictions.
          

          

            2. HTTPS Requirement: In production, the page must be loaded via HTTPS (or from localhost) to access the Bluetooth adapter. (Our live development workspace handles this automatically).
          

          

            3. Bluetooth Classic vs. BLE: Standard Web Bluetooth works strictly with Bluetooth Low Energy (BLE) modules (like HM-10, CC2541, HM-11, CC2640). If you are using classic modules (HC-05 or HC-06), they do not support GATT profiles natively and cannot connect over Web Bluetooth. Use an HM-10 module as a drop-in BLE replacement!
          

          

            4. Service Filtering: Ensure your Arduino script starts broadcasting standard FFE0 Serial, or Nordic NUS service, so this app's scanner can recognize and subscribe to it.

              Android Mobile Instructions
            

            

              Android supports Web Bluetooth natively, but system security requires strict permission steps:
            

            
    
              
  • You MUST use Google Chrome, Edge, or Opera.
    • 
              
  • Ensure your phone's system Location Services (GPS) is turned ON. Android requires Location to search for nearby physical Bluetooth beacons.
    • 
              
  • Go to Settings → Apps → Chrome → Permissions and ensure Nearby Devices (or Location and Bluetooth) are allowed.
    • 
              
  • Ensure the page is loaded over a secure HTTPS:// link.
    • 
            

          


          
          

            

              
              PWA Standalone App Advice
            

            

              If you installed this app to your phone's home screen, Web Bluetooth sandboxing permissions can occasionally get blocked by the OS. If connection fails, launch the app directly inside your standard mobile browser window instead.
